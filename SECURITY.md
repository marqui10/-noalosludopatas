# Política de Seguridad - No a los Ludópatas

Esta política de seguridad describe nuestro compromiso con la integridad de los datos, la transparencia de los algoritmos y la protección de los usuarios que visitan [marqui10.github.io/-noalosludopatas/](https://marqui10.github.io/-noalosludopatas/).

## 1. Compromiso Ético y Transparencia
El objetivo principal de este proyecto es la concienciación. Por ello, la seguridad se basa en la **transparencia**:
- **Algoritmos Abiertos:** Toda la lógica de probabilidades y simulación es pública y puede ser auditada en este repositorio.
- **Sin Manipulación:** Los sistemas de generación aleatoria están diseñados para reflejar realidades estadísticas, sin algoritmos de "recuperación" o "engaño" comunes en las plataformas de juego real.

## 2. Privacidad del Usuario
Para garantizar la seguridad de quienes buscan ayuda o información:
- **Cero Datos Personales:** Este sitio no solicita, almacena ni procesa nombres, correos electrónicos, números de teléfono ni datos financieros.
- **Navegación Cifrada:** El sitio se sirve exclusivamente a través de **HTTPS** mediante GitHub Pages.
- **Sin Rastreo:** No utilizamos cookies de terceros ni herramientas de análisis que identifiquen al usuario fuera de este dominio.

## 3. Seguridad del Código
- **Dependencias:** Utilizamos herramientas automatizadas (como Dependabot) para monitorizar y actualizar librerías que puedan tener vulnerabilidades conocidas.
- **Integridad:** Cualquier cambio en la lógica de simulación debe pasar por un proceso de revisión para evitar la introducción de código malicioso o engañoso.

## 4. Reporte de Vulnerabilidades
Si encuentras un error de seguridad, una vulnerabilidad técnica o un fallo en la lógica de probabilidades que pueda ser explotado, te pedimos que actúes de forma responsable:

1. **No abras una "Issue" pública** inmediatamente si el fallo permite comprometer datos o la integridad del sitio.
2. Por favor, reporta el hallazgo enviando un mensaje privado al administrador del repositorio o mediante el sistema de avisos de seguridad de GitHub.
3. Se dará respuesta a los reportes en un plazo máximo de 48-72 horas.

## 5. Exclusión de Responsabilidad
Este proyecto es una herramienta educativa. Aunque nos esforzamos por mantener el sitio 100% seguro y libre de errores, el usuario es responsable del uso que haga de la información aquí proporcionada.
